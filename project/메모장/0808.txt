301 영구적으로 uri값이 변경됨 추후 리소스 요청시 새로운 uri값 입력
302 일시적으로 uri 값이 변경됨 재 요청시 동일 uri 값 입력

402번 결제가 필요함 결제 시스템을 위해서 만들엇으나 사용을 안함
403번 컨텐츠 접근 ㄴ권한이 없음 
(401번 과 차이점은 서버가 클라이언트가 누군지 안다는 것)
404번 찾을 수 없음 리소스파일을 인식못함
405번 post로 받겠다는데 get요청을 한 경우 허용되지 않는 메소드

동적페이지 / 정적페이지 차이?
동적 페이지는 데이터를 스크립트를 이용해서가공하여 사용하는 페이지를 뜻하고
정적 페이지는 서버에 등록 되어 있는 파일만 출력
동적 페이지 장단점 비용이 상대적 비싸고 느리고
웹서버 말고도 웹 어플리케이션 서버 was를 사용해야 한다.

정적 페이지는 web서버만 이용해도 된다.
web서버는 정적 페이지를 클라이언트 에게 보여주고
WAS 웹 어플리케이션 서버는 db조회, 알고리즘 수행 등 바빠서
정적 페이지까지 처리하기가 좀 버겁다
web서버에서 WAS를 여러대 연결을 해서 사용도 가능하고
web 서버와 was서버를 구분하는 이유는 
1.로드 밸런싱 용으로 사용 (부하를 줄이는거지)
2.정적컨텐츠 등은 빠르게 web서버 에서 바로 처리를 해주는 거지
